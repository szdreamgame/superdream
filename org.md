## 🔄 工作流程

### 任务分配流程

```
潘总 → 超梦 → 对应智能体 → 执行 → Plaza 发布 → 潘总确认
```

### 协作流程

1. **任务接收**: 超梦接收潘总任务
2. **任务分析**: 超梦分析任务类型
3. **任务分配**: 分配给对应智能体
4. **执行**: 智能体执行任务
5. **发布**: 在 Plaza 发布结果
6. **确认**: 潘总确认结果

### 委派流程

智能体之间可以委派任务：

```
承道 → 棱镜：请帮忙整理这份技术文档
译文 → 画语：请为这个视频生成封面图
```

## 🤝 Agent 委派任务功能

### 委派协议

智能体间委派任务遵循以下协议：

```markdown
**委派请求**:
- 委派者：[智能体名称]
- 接收者：[智能体名称]
- 任务描述：[具体任务]
- 优先级：[high/medium/low]
- 截止时间：[可选]
- 依赖资源：[文件/数据/权限]

**接收响应**:
- ✅ 接受：确认接收，预计完成时间
- ⚠️ 有条件接受：需要额外资源/权限
- ❌ 拒绝：说明原因（忙碌/无能力/权限不足）
```

### 委派场景示例

| 场景 | 委派者 | 接收者 | 任务类型 |
|------|--------|--------|----------|
| 技术文档整理 | 承道 | 棱镜 | 文档处理 |
| 视频封面生成 | 译文 | 画语 | 视觉设计 |
| 商机报告翻译 | 御影 | 译文 | 翻译 |
| 数据分析可视化 | 超梦 | 棱镜 | 数据分析 |
| 技术方案配图 | 承道 | 画语 | 图片生成 |

### 委派权限矩阵

| 委派者 \ 接收者 | 超梦 | 承道 | 棱镜 | 译文 | 画语 | 御影 |
|-----------------|------|------|------|------|------|------|
| **超梦** | - | ✅ | ✅ | ✅ | ✅ | ✅ |
| **承道** | ⚠️ | - | ✅ | ⚠️ | ✅ | ❌ |
| **棱镜** | ⚠️ | ⚠️ | - | ✅ | ✅ | ❌ |
| **译文** | ⚠️ | ❌ | ✅ | - | ✅ | ❌ |
| **画语** | ⚠️ | ❌ | ✅ | ✅ | - | ❌ |
| **御影** | ⚠️ | ❌ | ✅ | ✅ | ✅ | - |

**图例**：
- ✅ = 可直接委派
- ⚠️ = 需要超梦协调
- ❌ = 不允许直接委派（需通过超梦）

### 委派任务追踪

所有委派任务自动记录到审计日志：

```markdown
### [AUDIT-YYYYMMDD-XXX] 任务委派

**时间**: ISO-8601 时间戳
**委派者**: 智能体 A
**接收者**: 智能体 B
**任务**: 任务描述
**状态**: pending → in_progress → completed
**耗时**: X 分钟
```

### 委派冲突解决

当出现委派冲突时（如接收者正在处理其他任务）：

1. **优先级判断**: high > medium > low
2. **时间戳判断**: 先到的任务优先
3. **超梦仲裁**: 无法自动解决时，上报超梦协调

```json

  "name": "[智能体名称]",
```

```markdown
# IDENTITY.md - [智能体名称]

- **性格**: [沟通风格描述]
```

### 入职评估标准

| 评估维度 | 权重 | 合格标准 |
|----------|------|----------|
| 技术能力 | 30% | 能独立完成分配的任务 |
| 协作能力 | 25% | 能与其他智能体有效协作 |
| 响应速度 | 20% | 平均响应时间 < 2 分钟 |
| 任务质量 | 15% | 任务完成率 > 90% |
| 学习能力 | 10% | 能快速掌握新技能 |

**总分 ≥ 80 分** = 通过入职评估

| 新智能体类型 | 推荐导师 |

| 生产力类 | 棱镜 |
| 内容类 | 译文 |
| 设计类 | 画语 |
| 商业类 | 御影 |

**导师职责**:
- 指导基础配置
- 解答技术问题
- 协助技能培训
- 评估入职表现
